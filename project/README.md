# LLM-ассистент для абитуриентов (RAG + FAQ + Structured Data)

## Описание проекта

Проект представляет собой интеллектуального ассистента для абитуриентов, который отвечает на вопросы о поступлении, образовательных программах, стоимости обучения, проходных баллах, количестве мест, требованиях к ЕГЭ и устройстве мегакластеров.

Система строится по принципу **RAG (Retrieval Augmented Generation)**: сначала находится релевантная информация в базе знаний, затем LLM формирует ответ **только на основе найденных данных**. Для вопросов с точными значениями используется отдельная обработка табличных данных.

Проект рассчитан на **команду до 4 человек**.  
Защита проекта проходит в формате **питчинга на последнем семинаре**, где каждая группа демонстрирует решение, архитектуру и примеры работы системы.

---

## Цель

Снизить нагрузку на сотрудников приёмной комиссии за счёт автоматизации ответов на типовые вопросы абитуриентов и их родителей.

Примеры вопросов:
- Какие ЕГЭ нужны для поступления на анализ данных и искусственный интеллект?
- Сколько стоит обучение на бизнес-информатике?
- Что такое мегакластер?
- Сколько бюджетных мест на программе?
- Чем отличаются мегакластеры «Экономика» и «Информационные технологии»?

---

## Что должна уметь система

- принимать вопрос на естественном языке;
- определять тип вопроса;
- искать релевантную информацию в базе знаний;
- извлекать точные значения из таблиц;
- генерировать понятный, краткий и корректный ответ;
- не выдумывать данные, если ответ не найден;
- фильтровать токсичные и некорректные запросы;
- работать через удобный интерфейс: Telegram-бот, веб-приложение или REST API.

## Датасеты проекта

### 1. `all_program.xlsx`
Основной датасет по образовательным программам.

Содержит информацию о:
- названии программы (`program`);
- мегакластере (`megacluster`);
- институте (`institute`);
- направлении подготовки (`major`);
- треках (`tracks`);
- форме и длительности обучения (`edu_form`, `edu_years`);
- проходных баллах (`pass_2024`);
- количестве бюджетных и платных мест (`budget_2025`, `contract_2025`);
- стоимости обучения (`cost`);
- описании программы (`desc`);
- навыках выпускника (`skills`);
- требованиях к ЕГЭ (`eges_contract`, `eges_budget`).

### 2. `Database-2.xlsx`
Текстовая база знаний.

Содержит:
- `header` — заголовок раздела;
- `text` — описание.

Подходит для ответов на общие вопросы:
- что такое мегакластер;
- кому подходит направление;
- кто партнёры;
- как устроена структура образовательных направлений.

### 3. `Database.xlsx`
FAQ-датасет.

Содержит:
- `Question`;
- `Answer`;
- `Question type`.

Используется для:
- поиска готовых ответов;
- FAQ-режима;
- классификации типов вопросов;
- проверки качества retrieval.

### 4. `ru_abusive_words.txt`
Список нежелательных слов для базовой модерации.

### 5. `ru_curse_words.txt`
Дополнительный словарь нецензурной лексики.

## Метрики качества

Для демонстрации на защите желательно показать не только интерфейс, но и оценку качества.

### Что можно измерить
- accuracy FAQ retrieval;
- top-k retrieval quality;
- долю корректных ответов по таблице;
- долю случаев, когда система честно отвечает “не найдено”;
- latency одного ответа;
- долю токсичных запросов, которые были корректно отфильтрованы.

### Минимальный eval-набор
Соберите 20–40 контрольных вопросов:
- FAQ;
- стоимость;
- баллы;
- ЕГЭ;
- общие вопросы;
- сложные формулировки;
- токсичные запросы.

---

## Ограничения и риски

- Нельзя позволять LLM выдумывать стоимость, баллы и ЕГЭ.
- Нужно обрабатывать разные формулировки одной программы.
- В `pass_2024` могут быть не только числа, но и текстовые значения.
- Не все поля заполнены полностью, поэтому нужен fallback.
- Для продвинутых ответов лучше использовать hybrid retrieval.
- Словари токсичных слов дают только базовую модерацию и могут давать false positive / false negative.

---

## Минимальные требования к результату

Проект считается завершённым, если система:
- принимает вопрос пользователя;
- умеет отвечать минимум на 3 типа запросов:
  - FAQ,
  - табличные запросы,
  - общие текстовые вопросы;
- использует retrieval;
- подключает LLM;
- фильтрует токсичные сообщения;
- имеет интерфейс;
- готова к демонстрации на питчинге.
